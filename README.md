# FutureMatch AI

AI-powered career guidance and job-matching platform connecting TVET students in Kenya with careers, internships, jobs, and learning opportunities — matched to their skills, interests, and current labour-market demand.

Built with **HTML5, CSS3, vanilla JavaScript (ES6), Bootstrap 5, Font Awesome and Chart.js**. No backend or build step required — everything runs from local JSON data and `localStorage`.

## Getting Started

No installation needed. Simply open `index.html` in a browser, or serve the folder with any static server:

```bash
# Option 1: just open the file
open index.html

# Option 2: serve locally (recommended for consistent relative paths)
npx serve .
# or
python3 -m http.server 8080
```

Then visit `http://localhost:8080` (or your chosen port).

## How the "AI" works

Since there is no backend, `js/ai.js` contains a deterministic **simulated recommendation engine**:

- Careers are scored against a student's **skills (40%)**, **interests (30%)**, **course relevance (15%)**, and **assessment answers (15%)**.
- The skill-assessment quiz tags each answer to specific careers; tag frequency feeds back into the scoring model.
- A learning roadmap is generated by comparing the top-matched career's required skills against the student's existing skills.

This produces realistic, varied, and personalised-feeling results without any external API calls.

## Data & Persistence

- All static content (careers, jobs, testimonials, team, quiz questions) lives in `js/data.js`.
- Student sessions, profiles, and quiz results are stored in the browser's `localStorage` (see `FMAuth` in `js/app.js`), so registering → taking the assessment → viewing the dashboard is a fully connected flow.
- Saved/bookmarked jobs persist under the `futurematch_saved_jobs` key.

## Notes for Developers

- No build tools required — all dependencies (Bootstrap, Font Awesome, Chart.js, Google Fonts) are loaded via CDN.
- To connect a real backend later, replace the contents of `js/data.js` with API calls and swap `FMAuth`'s localStorage calls for real authentication/session handling.
- `js/ai.js` is structured so its scoring weights and logic can be swapped for a real ML/recommendation service without changing any UI code — every page calls it through the `FutureMatchAI_Engine` object.
